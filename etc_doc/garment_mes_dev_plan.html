▼
원칙 1 — ERP8는 유일한 마스터 원천, MES9는 실행 레이어. 스타일·BOM10·자재 코드는 ERP에서 받아 사용, MES가 독자 생성 금지.
원칙 2 — 데이터는 Layer A~D로 엄격히 구분. ERP 수신 데이터(Layer A)는 정리 가능, MES 생성 실행 기록(Layer C)은 영구 보관.
원칙 3 — 팀(라인) 단위 실적이 Phase 1 기본. 작업자 개인 추적은 Phase 2에서 자발적 전환. MES는 임금 계산 절대 금지 (ADR11-005).
원칙 4 — ERP API 합의가 모든 것의 전제조건. W1 Day1에 ERP 담당자와 기술 협의 착수. 미수령 시 Mock ERP로 개발 진행, W13으로 실 연동 이동.

각주
1Raza & Bois: Claude Code를 대규모 프로젝트에 효과적으로 적용하기 위한 운영 방법론. CLAUDE.md·Plan Mode·Stream A/B 병렬 개발 등을 포함.
2Harness: 7개 자동화 검증 도구 집합. PR마다 실행되어 계약 위반·성능 회귀·도메인 규칙 위반을 자동 차단.
3Plan Mode: Claude Code가 코드를 작성하기 전에 구현 계획을 먼저 제시하고 승인을 받는 단계.
4Code Mode: Plan Mode 승인 후 실제 코드를 생성·수정하는 단계.
5PR (Pull Request): 코드 변경사항을 메인 브랜치에 합치기 위한 검토 요청.
6머지 (Merge): PR 검토 완료 후 코드를 메인 브랜치에 통합하는 행위.
7기술 부채 (Technical Debt): 빠른 개발을 위해 임시방편으로 만든 코드가 나중에 유지보수 비용을 증가시키는 현상.
8ERP (Enterprise Resource Planning): 전사적 자원 관리 시스템. 오더·재무·인사 등을 통합 관리.
9MES (Manufacturing Execution System): 제조실행시스템. 생산 현장의 실행·추적·품질을 관리.
10BOM (Bill of Materials): 자재 명세서. 제품을 만들기 위한 부품·원자재 목록.
11ADR (Architecture Decision Record): 아키텍처 결정 기록. 결정 날짜·맥락·이유·결과를 영구 보존.
12ISA-95: ERP와 공장 현장 시스템 간 공통 언어를 정의하는 국제 표준 (IEC 62264).
13LOT: 동일 조건(오더·색상)으로 생산되는 의류 묶음 단위. Roll→LOT→Bundle→Carton→B/L 추적.

# scripts/domain-validate.sh
CRITICAL=0

check_forbidden() {
result=$(grep -rn "$2" apps/api/src --include="*.ts" | grep -v ".test.")
[ -n "$result" ] && { echo "❌ $1"; echo "$result"; CRITICAL=$((CRITICAL+1)); }
}

check_forbidden "C-1: AQL 하드코딩" "dhuThreshold.*[0-9]\."
check_forbidden "C-2: 임금계산" "wage\|salary\|payroll"

check_forbidden "C-4: Zod 없는 외부입력" "req\.body\b"

[ $CRITICAL -eq 0 ] && echo "✅ @domain-validator 통과 (0건)"

async aggregateDailyLine(date: string) {
for (const line of await this.lineRepo.findActive()) {
const outputs = await this.lineOutputRepo.findByLineAndDate(line.id, date)
if (!outputs.length) continue

const totalOutput = outputs.reduce((s,o) => s+o.outputQty, 0)
const totalDefect = outputs.reduce((s,o) => s+o.defectQty, 0)

lineId: line.id, workDate: date,
totalOutput, totalDefect,

entityType: 'LINE_PRODUCTION_RESULT',
payload: { line_code: line.lineCode, work_date: date,
total_output: totalOutput, total_defect: totalDefect }
})
}
}
